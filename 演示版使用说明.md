# 黑卫士 AI 数字档案管理系统 1.0 演示版说明

## 版本定位

这是“可安装/可演示版本”，用于演示系统界面、检索入口、作品类型分类、权限展示、媒体预览、待收尾作品池、界面自定义能力和本机只读索引闭环。

当前版本可以使用模拟数据，也可以只读扫描本机样本目录并生成真实索引。扫描不移动、不删除、不改名真实文件，不连接外部数据库。

多源存储已经纳入第一版方案：本机硬盘、在线硬盘、外接硬盘、U盘、云端云盘、公有云、私有云、局域网网盘、NAS、不同主机硬盘、同主机多接口硬盘，都先统一做只读索引和交叉校验，不急着复制 40T-60T 原始大文件。

新的归集主线是：手机、iPad、旧电脑、邮箱、云端、NAS、U盘、SD卡、相机、摄像机都要纳入统一管理。最终以 NAS 作为归总硬件，完整采集必须先进入 NAS 暂存区，再分类、去重、索引和人工复核。第一阶段仍然不擅自移动、删除、改名真实文件；只有授权后才做完整采集。

图片资料后续可以进入人脸识别候选池，帮助按人物、活动、项目查找，但必须受权限控制。未授权图片只做文件级索引，不做人脸识别。

## 启动方式

推荐方式：直接在桌面双击：

```text
黑卫士AI数字档案管理系统.command
```

进度看板入口：

```text
黑卫士AI项目进度看板.command
```

只读扫描文件夹入口：

```text
黑卫士AI只读扫描文件夹.command
```

项目文件入口：

```text
黑卫士AI数字档案管理系统-项目文件
```

桌面主系统和看板入口会自动判断本地服务是否已经启动。已启动就直接打开；未启动就自动启动后打开页面。只读扫描入口会让你选择一个文件夹，只读取文件信息并更新索引，不移动、不删除、不改名真实文件。

在项目目录运行：

```bash
npm run dev
```

然后打开终端里显示的地址。默认是：

```text
http://127.0.0.1:4173/
```

如果 4173 端口已经被占用，系统会自动尝试 4174、4175 等后续端口，并在终端里提示最终地址。

也可以直接打开：

```text
界面原型-v1/index.html
```

推荐用 `npm run dev`，因为演示时路径和脚本加载更稳定。

## 自检命令

```bash
npm test
```

自检会确认：

- 演示版关键文件完整
- 页面不依赖外网图标库
- 项目/组织/人员检索入口存在
- 作品类型检索入口存在
- 本机索引快搜入口存在
- 公司类型、周期阶段、作品等级、L0-L6 密级规则存在
- 存储来源、接入方式、同步策略、交叉校验、存储风险、成本等级存在
- NAS 暂存区、采集授权、去重状态、人脸识别候选权限口径存在
- 方案、标书、合同、邮件、Excel、标识文件等样本存在

## 单机索引命令

扫描当前项目目录：

```bash
npm run scan
```

扫描指定样本目录：

```bash
npm run scan -- /要扫描的文件夹路径
```

扫描是只读操作，不移动、不删除、不改名真实文件。扫描后会生成：

```text
界面原型-v1/archive-index.json
界面原型-v1/archive-index-data.js
```

页面会优先读取本机扫描索引；没有索引时保留演示数据。

## 单机验收与备份

生成单机验收报告：

```bash
npm run verify
```

生成索引和看板快照：

```bash
npm run backup
```

检查本地服务、首页、看板和脱敏索引：

```bash
npm run health
```

命令行也可以直接打开：

```bash
npm run open
npm run board
npm run scan:choose
```

快照只复制索引、看板和验收报告，不移动、不删除真实档案。

## 演示路线

1. 打开大屏首页，看文件总量、全文索引、待收尾作品、备份健康。
2. 使用主搜索框输入“发布会”或“投标”，查看结果变化。
3. 在项目/组织/人员检索里输入：
   - 公司名称：广告公司
   - 部门名称：策划部
   - 员工姓名：张三
4. 在作品类型检索里点击：
   - 方案/提案
   - 标书/投标文件
   - 邮件
   - Excel/电子表格
   - 标识/LOGO
5. 在格式/扩展名里输入：
   - PSD
   - EML
   - XLSX
   - PDF
6. 点击结果行，观察右侧媒体预览和元数据变化。
7. 打开界面设置，演示：
   - 更换主题色
   - 自定义主色、侧栏色、背景色
   - 切换字体
   - 调整字号
8. 查看待收尾作品池，说明后续 AI 补全和评分机制。
9. 在项目/组织/人员检索里输入“手机”“iPad”“旧电脑”“邮箱”“SD卡”“相机”“摄像机”“NAS 暂存区”，演示多来源归集入口。
10. 再输入“本机硬盘”“NAS”“云盘”“只读索引”“交叉校验”，演示多源存储和同步策略。
11. 打开右侧预览，查看存储来源、存储名称、接入方式、同步策略、同步状态、交叉校验、存储风险和成本等级。
12. 说明后续完整采集流程：授权后进入 NAS 暂存区，再分类、去重、索引；图片进入人脸识别候选池前必须做权限确认。

已完成：

- 大屏首页
- 智能检索
- 项目/公司/部门/员工/职能检索
- 作品类型检索

- 媒体预览区
- 待收尾作品池
- 评分、密级、负责人、周期展示
- 公司类型、公司名称、部门建制、项目类型、负责人等级展示
- 模糊时间段的周期搜索和周期阶段搜索
- L0-L6 密级规则
- 完整状态和作品等级规则
- 界面颜色、字体、字号自定义
- 本地离线图标
- 本地演示服务
- 本机只读扫描、本机索引快搜、单机验收报告、备份快照
- 服务健康检查：确认首页、看板、健康接口和浏览器脱敏索引正常
- 多源存储字段：存储来源、存储名称、接入方式、同步策略、同步状态、交叉校验、存储风险、成本等级
- 多源接入方案：手机、iPad、旧电脑、邮箱、本机硬盘、在线硬盘、外接硬盘、U盘、SD卡、相机、摄像机、云端云盘、公有云、私有云、局域网网盘、NAS、不同主机硬盘、同主机多接口硬盘
- NAS 暂存区口径：授权后完整采集先进入暂存区，再分类、去重、索引和复核

未完成：

- 真实 40T-60T 硬盘全量接入
- 文件上传和入库
- OCR、录音转写、视频抽帧
- 真实图片人脸识别候选和权限审批
- 登录和真实权限系统
- 数据库保存
- 真实文件预览、下载、复制审批
- AI 自动补全和真实知识库

安全边界：

- L4、L5、L6 默认不进公有云，不进入 AI 训练。
- 第一阶段默认只读索引，不移动、不删除、不改名真实文件。
- 手机、iPad、旧电脑、邮箱、U盘、SD卡、相机、摄像机等来源，必须授权后才完整采集到 NAS 暂存区。
- 图片进入人脸识别候选池前必须有权限控制。
- 云本同步只建议用于 L0-L2 或已授权资料；NAS 和私有云更适合中高密级资料。

## 下一阶段：1 天单机可用版

建议下一步只做单机闭环，不做多人系统：

1. 本机选择一个文件夹作为样本档案库。
2. 扫描文件名、路径、大小、格式、修改时间。
3. 生成本地 JSON 或 SQLite 索引。
4. 在当前界面里读取真实索引。
5. 支持真实文件打开位置。
6. 支持按格式、公司、项目、关键词初步分类。
7. 支持按存储来源、接入方式、同步策略和风险等级筛选。
8. 保留模拟数据开关，方便继续演示。
9. 补齐 NAS 暂存区字段，为后续完整采集、去重和图片人脸识别候选做准备。
