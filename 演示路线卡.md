# 黑卫士 AI 数字档案管理系统 3 分钟演示路线卡

## 演示目标

让观看者在 3 分钟内看懂三件事：系统能启动、能检索真实本机索引、能按后续真实硬盘目录和多源存储继续扩展。

补充讲解主线：手机、iPad、旧电脑、邮箱、云端、NAS、U盘、SD卡、相机、摄像机都要能归集，最终以 NAS 作为归总硬件。第一阶段只读摸底，不擅自移动、删除、改名真实文件；授权后才完整采集到 NAS 暂存区，再分类、去重、索引。

## 启动

推荐一键演示：

```bash
npm run demo
```

如果只想启动页面：

```bash
npm run dev
```

打开终端提示的地址，一般是：

```text
http://127.0.0.1:4173/
```

## 3 分钟演示路线

| 时间 | 操作 | 讲解重点 |
|---|---|---|
| 0:00-0:30 | 打开首页 | 这是公司 26 年数字资产的大屏入口，左侧是一级菜单，中间是检索和资产概览 |
| 0:30-1:00 | 看“本机索引文件”指标 | 当前页面已经读取本机真实索引，不只是静态 Demo |
| 1:00-1:30 | 在“本机索引快搜”输入“项目进度” | 能按文件名、相对路径、格式快速定位本机文件 |
| 1:30-2:00 | 点击“程序文件”快捷按钮 | 能按文件类型快速筛选，后续可扩展图片、视频、录音、合同、标书 |
| 2:00-2:30 | 点击一条结果，看右侧预览 | 右侧展示相对路径、大小、修改时间、密级、评分、负责人、存储来源和同步策略 |
| 2:30-2:45 | 搜索“NAS 暂存区”或“SD卡” | 说明后续手机、旧电脑、邮箱、相机、摄像机等来源先归集到 NAS 暂存区 |
| 2:45-3:00 | 打开“项目进度看板” | 用甘特图和进度表看当前开发节奏，不需要反复追问 |

## 重点演示入口

- 主搜索：适合查项目、公司、作者、内容关键词。
- 项目/组织/人员检索：适合按项目名称、公司名称、部门名称、员工姓名、人事职能查。
- 作品类型检索：适合查图片、视频、录音、邮件、合同、PPT、Word、Excel、扫描件、标识文件、方案、标书。
- 本机索引快搜：适合快速查当前已经扫描出来的真实本机文件。
- 多源存储检索：适合按手机、iPad、旧电脑、邮箱、本机硬盘、在线硬盘、外接硬盘、U盘、SD卡、相机、摄像机、云端云盘、公有云、私有云、局域网网盘、NAS、不同主机硬盘、同主机多接口硬盘查。
- 同步策略检索：适合查只读索引、NAS 暂存区、云本同步、NAS 主库、私有云同步、公有云冷备、交叉校验。

## 验收口径

- 能启动：`npm run dev` 或 `npm run demo` 正常启动。
- 能自检：`npm test` 通过。
- 能验收：`npm run verify` 生成单机版验收报告。
- 能备份：`npm run backup` 生成索引和看板快照。
- 能检索：页面显示本机索引文件，并且“本机索引快搜”可以筛选和清空。
- 能看多源字段：预览区显示存储来源、存储名称、接入方式、同步策略、同步状态、交叉校验、存储风险和成本等级。

## 当前边界

当前版本是单机演示和单机索引闭环，不会移动、不删除、不改名真实文件。真实 40T-60T 硬盘接入前，需要先按 SOP 选择样本目录，小范围跑通后再扩大扫描范围。

L4、L5、L6 默认不进公有云，不进入 AI 训练；云本同步只建议用于 L0-L2 或已授权资料。

完整采集到 NAS 暂存区必须先授权。图片后续可以进入人脸识别候选池，但必须有权限控制；未授权图片只做文件级索引。
